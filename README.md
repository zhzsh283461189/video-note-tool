# 视频笔记工具 (Video Note Tool)

一个功能强大的纯前端视频学习笔记应用，支持视频播放、截图标注、富文本编辑和多种导出格式。专为学习、研究和内容创作场景设计，提供完整的视频笔记解决方案。

## ✨ 核心特性

### 📹 视频播放与管理
- 支持本地视频文件加载与播放（MP4、WebM等格式）
- 智能内存管理，自动释放旧的视频URL对象
- 响应式视频播放器，适配不同屏幕尺寸

### 📸 智能截图系统
- **即时截图**：一键截取视频当前帧并复制到剪贴板
- **截图标注**：打开专业标注界面，支持多种标注工具
- **智能存储**：自动保存到LocalStorage，支持过期清理（1小时）
- **快速插入**：一键将截图插入笔记编辑器

### ✏️ 专业标注工具
- **形状标注**：矩形框、圆形框、箭头指示
- **文字标注**：支持自定义字体大小（小/中/大）
- **移动工具**：可拖动调整已绘制的标注位置
- **样式定制**：
  - 8种颜色选择（红、绿、蓝、橙、黄、青、紫、灰）
  - 3种线条粗细（细/中/粗）
  - 3种字体大小（16px/24px/32px）
- **撤销功能**：支持逐步撤销标注操作
- **快捷键支持**：Esc取消、Enter完成、Ctrl+Z撤销

### 📝 富文本编辑器
- 基于Quill.js的强大富文本编辑器
- 支持格式化文本（粗体、斜体、下划线、删除线）
- 多级标题（H1、H2、H3）
- 列表功能（有序列表、无序列表）
- 图片插入与管理
- 时间戳快速插入（格式：[HH:MM:SS]）
- 友好的占位提示和使用指南

### 💾 数据管理
- **手动保存**：导出为JSON文件，包含完整笔记数据和元信息
- **导入恢复**：从JSON文件恢复笔记内容和视频信息
- **自动备份**：每30秒自动保存到LocalStorage
- **智能检测**：启动时检测未保存的笔记并提示恢复
- **容量管理**：自动检测存储空间，超过4MB时智能提示

### 📄 多格式导出
- **Word导出**：使用html-docx-js生成标准.docx文档
- **智能排版**：
  - 自动限制图片宽度为590px（适应A4纸张）
  - 添加文档标题、视频信息和导出时间
  - 专业的样式设计（边距、阴影、圆角）
- **PDF友好**：导出的Word文档可直接转换为PDF

### 📱 响应式设计
- **桌面端优化**：充分利用大屏幕空间
- **移动端适配**：专门的移动端CSS样式
- **横竖屏切换**：标注界面自动适应屏幕方向
- **触摸支持**：完整的触摸事件处理

## 📖 使用指南

### 基本工作流程

1. **加载视频**
   - 点击 "📁 视频" 按钮
   - 选择本地视频文件
   - 等待视频加载完成

2. **观看与截图**
   - 播放视频到关键位置
   - 点击 "📸 截图" 快速截图
   - 或点击 "✏️ 截图标注" 进行标注

3. **标注截图**（可选）
   - 选择标注工具（矩形/圆形/箭头/文字）
   - 选择颜色和线条粗细
   - 在截图上绘制标注
   - 使用移动工具调整位置
   - 按 Enter 或点击 "✓ 完成" 保存

4. **编写笔记**
   - 点击 "📋 插入截图" 将截图插入笔记
   - 点击 "⏰ 时间" 插入当前时间点
   - 使用富文本工具格式化内容
   - 添加个人思考和总结

5. **保存与导出**
   - 点击 "💾 保存" 导出JSON备份
   - 点击 "📄 Word" 导出Word文档
   - 系统每30秒自动备份

### 标注工具详解

#### 形状工具
- **矩形 (□)**：用于框选重点区域
- **圆形 (○)**：用于标记特定点或区域
- **箭头 (→)**：用于指示方向或关系

#### 文字工具 (A)
- 点击画布任意位置输入文字
- 支持三种字体大小：小(16px)、中(24px)、大(32px)
- 按 Enter 或失去焦点完成输入

#### 移动工具 (✥)
- 点击已绘制的标注
- 拖动到新位置
- 松开鼠标完成移动

#### 颜色选择
- 基础色：红、绿、蓝、橙、黄
- 扩展色（横屏模式）：青、紫、灰
- 默认颜色：红色

#### 线条粗细
- **细**：2px，适合精细标注
- **中**：4px，默认选项
- **粗**：6px，适合强调重点

### 快捷键

在标注界面可使用以下快捷键：
- `Esc`：取消标注，关闭界面
- `Enter`：完成标注，保存并关闭
- `Ctrl+Z` / `Cmd+Z`：撤销上一步操作

## 🏗️ 技术架构

### 前端技术栈
- **核心**：原生 HTML5 + CSS3 + JavaScript (ES5)
- **富文本编辑器**：Quill.js 1.3.6
- **文档导出**：html-docx-js, FileSaver.js 2.0.5
- **CDN资源**：所有第三方库通过CDN加载，无需本地安装

### 模块化设计

```
js/
├── main.js          # 应用入口，初始化和事件协调
├── video.js         # 视频管理模块（加载、播放、截图）
├── editor.js        # 编辑器模块（Quill初始化、内容操作）
├── biaoZhu.js       # 标注模块（Canvas绘图、工具管理）
├── save-load.js     # 数据管理模块（保存、导入、自动备份）
├── export.js        # 导出模块（Word生成、图片处理）
└── utils.js         # 工具函数（时间格式化、Toast提示、存储管理）
```

### 数据存储策略

1. **LocalStorage**
   - 截图缓存（videoNoteScreenshot）
   - 截图时间戳（videoNoteScreenshotTime）
   - 自动备份（videoNoteAutosave）
   - 容量限制：约4-5MB

2. **文件导出**
   - JSON格式：完整笔记数据（含视频名、创建时间、编辑时间、HTML内容）
   - DOCX格式：格式化的Word文档

3. **内存管理**
   - 视频URL对象自动释放
   - 截图1小时过期清理
   - Canvas资源及时清理

## ⚠️ 注意事项

### 数据安全
1. **视频文件**：仅在浏览器本地播放，不会上传到任何服务器
2. **笔记数据**：存储在本地，清除浏览器数据会丢失自动备份
3. **定期保存**：建议频繁手动保存为JSON文件
4. **隐私保护**：所有数据处理均在本地完成

### 浏览器兼容性
- ✅ Chrome 60+（推荐）
- ✅ Firefox 55+
- ✅ Safari 11+
- ✅ Edge 79+
- ⚠️ IE 不支持（使用了现代API）

**必需支持的API**：
- Clipboard API（截图复制功能）
- Canvas API（标注功能）
- LocalStorage（数据保存）
- FileReader（文件导入）

### 性能优化
1. **大文件处理**：
   - 笔记超过4MB时禁用自动保存
   - Word导出时图片过多会显示警告
   - 建议单次笔记不超过50张图片

2. **内存管理**：
   - 视频URL及时释放
   - 截图1小时自动清理
   - Canvas元素使用后清理

3. **存储空间**：
   - LocalStorage限制约5-10MB
   - 超过限制时提示手动保存
   - 自动清除过期截图

### 已知限制
1. 截图功能需要HTTPS或localhost环境（Clipboard API要求）
2. Word导出在大图片较多时可能较慢
3. 移动端标注精度受屏幕尺寸影响
4. 不支持在线视频（仅本地文件）

## 🐛 故障排除

### 常见问题

**Q: 截图后无法插入笔记？**
A: 检查是否点击了"插入截图"按钮，确保光标位置正确。

**Q: 自动保存失败？**
A: 笔记可能过大，请手动保存为JSON文件。

**Q: Word导出失败？**
A: 检查图片数量，减少截图后重试；或尝试分多次导出。

**Q: 标注工具不响应？**
A: 确保标注界面完全加载，刷新页面重试。

**Q: 视频无法播放？**
A: 检查视频格式是否为浏览器支持的格式（MP4/WebM）。

### 调试技巧

打开浏览器开发者工具（F12）：
- **Console**：查看详细日志和错误信息
- **Application**：查看LocalStorage中的数据
- **Network**：检查CDN资源加载情况
